# Style rules for this project

The code must always look like it was written by a human, not generated.

- No em-dashes (—), en-dashes (–), or unicode separator runs (e.g. `// ── section ──`).
- No ellipsis character (…); use `...` if needed.
- No fancy quotes (" " ' '); use straight quotes.
- No AI-style section banners or decorative comment headers.
- Comments only when they explain a non-obvious WHY. Never restate WHAT the code does.
- No trailing summary comments or "Generated by" footers.
- Prefer plain ASCII everywhere.
